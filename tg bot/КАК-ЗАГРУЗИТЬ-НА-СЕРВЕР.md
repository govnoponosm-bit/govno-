# Как загрузить бота на сервер — подробно для начинающих

## Что такое «сервер» простыми словами

Сейчас бот работает **только пока открыт терминал на вашем компьютере**. Выключили ПК — бот молчит.

**Сервер** — это другой компьютер в интернете, который работает **24 часа в сутки**. Вы загружаете туда папку с ботом, вставляете токен — и бот отвечает в Telegram всегда.

---

## Что вам понадобится

1. **Токен бота** — уже есть в `.env` (от @BotFather)
2. **Ваш Telegram ID** (число) — в `.env` как `ADMIN_IDS`
3. **VPS** — арендованный сервер (~200–400 ₽/мес), например:
   - [Timeweb](https://timeweb.com)
   - [Aeza](https://aeza.net)
   - [Selectel](https://selectel.ru)

При заказе выберите: **Ubuntu 22.04** или **24.04**, самый дешёвый тариф.

После оплаты вам придут на почту:
- **IP-адрес** — например `185.123.45.67`
- **Логин** — обычно `root`
- **Пароль** — для входа

## ЧАСТЬ 1. Подготовка на вашем компьютере

### Шаг 1. Проверьте файл `.env`

Откройте `c:\Users\anyab\OneDrive\Документы\cursor\tg bot\.env`

Должно быть **так** (пример):

```
BOT_TOKEN=7123456789:AAHxxxxxxxxxxxxxxxx
ADMIN_IDS=123456789
```

- `BOT_TOKEN` — от BotFather
- `ADMIN_IDS` — **только цифры** (узнать: @userinfobot → Start)

**Не** пишите `@username` в ADMIN_IDS — не сработает.

### Шаг 2. Скачайте программу для загрузки файлов

**WinSCP** (бесплатно): https://winscp.net/eng/download.php

Установите как обычную программу.

---

## ЧАСТЬ 2. Загрузка бота на сервер

### Шаг 3. Подключитесь к серверу в WinSCP

1. Запустите **WinSCP**
2. Новое подключение:
   - **Протокол:** SFTP
   - **Имя хоста:** ваш IP (из письма хостинга)
   - **Порт:** 22
   - **Имя пользователя:** `root`
   - **Пароль:** из письма
3. Нажмите **Войти** → при вопросе о ключе — **Да**

Слева — ваш компьютер, справа — сервер.

### Шаг 4. Создайте папку на сервере

Справа (сервер):
1. Зайдите в `/opt` (двойной клик)
2. Правой кнопкой → **Создать** → **Каталог** → имя: `tg-bot`

### Шаг 5. Скопируйте файлы бота

Слева откройте папку:
`C:\Users\anyab\OneDrive\Документы\cursor\tg bot`

Выделите и перетащите **на сервер** в `/opt/tg-bot/`:

- `bot.py`
- `config.py`
- `database.py`
- `keyboards.py`
- `middlewares.py`
- `states.py`
- `requirements.txt`
- `.env`  ← **обязательно!**
- `.env.example`
- папки `handlers` и `services`
- папку `deploy`

**НЕ копируйте:**
- папку `.venv` (большая, не нужна)
- папку `__pycache__` (если есть)

`bot.db` — можно скопировать, если хотите сохранить фильмы и пользователей. Если нет — создастся новая база.

---

## ЧАСТЬ 3. Запуск бота на сервере

### Шаг 6. Откройте терминал к серверу

В WinSCP: меню **Команды** → **Открыть терминал** (или Ctrl+T)

Откроется чёрное окно — вы «внутри» сервера.

### Шаг 7. Установите и запустите бота

Скопируйте команды **по одной**, вставьте в терминал, Enter:

```bash
cd /opt/tg-bot
```

```bash
chmod +x deploy/install.sh
```

```bash
./deploy/install.sh
```

Подождите 1–3 минуты. В конце должно быть что-то вроде `Бот запущен!`

### Шаг 8. Проверка

```bash
systemctl status tgbot
```

Должно быть зелёное **active (running)**.

Откройте бота в Telegram → `/start` — должен ответить.

### Шаг 9. Выключите бота на компьютере

На **вашем ПК** в терминале Cursor нажмите **Ctrl+C**, если бот там ещё запущен.

**Один токен = один запуск.** Иначе будут сбои.

## Полезные команды на сервере

| Что сделать | Команда |
|-------------|---------|
| Перезапустить бота | `systemctl restart tgbot` |
| Остановить | `systemctl stop tgbot` |
| Запустить | `systemctl start tgbot` |
| Посмотреть логи | `journalctl -u tgbot -n 50` |
| Статус | `systemctl status tgbot` |

## Если что-то не работает

### «Connection refused» в WinSCP
- Проверьте IP и пароль
- Подождите 5–10 мин после создания VPS

### Бот не отвечает в Telegram
1. `systemctl status tgbot` — работает ли?
2. `journalctl -u tgbot -n 30` — есть ли ошибки?
3. Проверьте `.env` на сервере — есть ли BOT_TOKEN?
4. Бот на ПК выключен?

### Ошибка ADMIN_IDS
В `.env` только цифры: `ADMIN_IDS=123456789`

### Обновили код на ПК
Снова скопируйте файлы через WinSCP, затем:
```bash
cd /opt/tg-bot
systemctl restart tgbot
```

## Альтернатива без сервера

Если не хотите платить за VPS — бот может работать **только с вашего ПК**, пока он включён и запущен:

```powershell
cd "C:\Users\anyab\OneDrive\Документы\cursor\tg bot"
.\.venv\Scripts\python.exe bot.py
```

Это не «сервер», но для тестов часто достаточно.

## Краткая схема

```
1. Купить VPS (Ubuntu)
2. WinSCP → подключиться по IP
3. Скопировать папку бота в /opt/tg-bot (с .env)
4. Терминал → ./deploy/install.sh
5. Выключить бота на ПК
6. Проверить в Telegram
```

Готово.
